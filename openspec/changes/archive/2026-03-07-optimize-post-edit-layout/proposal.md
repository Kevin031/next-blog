# 提案：优化文章编辑页面布局

## 为什么

当前文章编辑页面采用垂直堆叠布局，所有字段使用 18 列宽度，在大屏幕上左右空间浪费严重。编辑器固定 500px 高度，无法充分利用屏幕空间，长文档编辑体验不佳。需要优化布局以提高空间利用率，改善编辑体验，并预留未来扩展能力。

## 变更内容

### 布局重组

- **顶部固定栏**：将标题输入框、返回按钮和保存按钮整合到顶部固定区域，使操作始终可见
- **工具栏**：将标签选择器和编辑器类型选择器整合到第二行工具栏，采用左右布局
- **编辑器区域**：移除固定高度限制，使用最小高度 600px 并自适应屏幕空间
- **状态栏**：将可见性开关移至底部状态栏，提供更直观的状态显示

### 交互优化

- 添加 `Ctrl+S` / `Cmd+S` 快捷键保存功能
- 标题输入框增加字符计数显示（50/100）
- 支持全屏编辑模式（纯前端实现）
- 编辑器高度自适应窗口变化

### 响应式设计

- 桌面版（> 1200px）：完整布局
- 平板版（768px - 1200px）：压缩布局，简化工具栏
- 移动版（< 768px）：垂直堆叠，简化交互

### 预留扩展

- 工具栏和状态栏提供插槽接口，支持未来插入新组件
- 响应式布局设计，支持未来添加新的功能区块

### 数据层

- **不增加任何数据库字段**
- 仅使用现有字段：`title`、`tags`、`content_type`、`content`、`visible`
- 启用但未使用的 `author` 字段暂不添加到 UI，预留未来扩展

## 功能 (Capabilities)

### 新增功能

（无新增业务功能，此次变更主要为布局和交互优化）

### 修改功能

- `post-management`: 修改文章编辑页面的布局需求，从垂直堆叠改为分层布局，增加快捷键、字符计数、全屏模式等交互功能，优化响应式体验。

## 影响

### 前端代码

- `admin-web/src/views/posts/edit.vue`：完全重构布局结构
- `admin-web/src/components/`：可能需要创建新的布局组件（TopBar、Toolbar、StatusBar）
- 不涉及新的 API 调用或数据模型变更

### 后端代码

- 无需修改
- 使用现有 API 接口和数据结构

### 数据库

- 无需变更
- 使用现有 PostEntity 字段

### 用户体验

- 编辑器获得更大空间，提升写作体验
- 常用操作始终可见，减少滚动
- 快捷键支持，提高效率
- 移动端友好，支持多设备访问

### 依赖项

- 使用现有 md-editor-v3 和 Element Plus 组件
- 无需新增依赖
